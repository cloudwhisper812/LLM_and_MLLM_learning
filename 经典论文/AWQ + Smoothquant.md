# AWQ (Activation-aware Weight Quantization): 保护 1% 精英的 INT4 量化艺术

## 1. 背景：解决的是什么问题？算法核心
- 大模型的权重分布是极度不平等的，存在大约1% 的“显著权重（Salient Weights）”，它们决定了模型 99% 的智商。如果用传统的 Round-To-Nearest (RTN) 把所有权重一视同仁地硬切成 INT4，这 1% 的关键权重会产生巨大的舍入误差，直接导致模型输出崩溃。
- AWQ 的灵魂： 权重的“显著性”不由它自己的数值大小决定，而完全取决于它对应的输入激活值（Activation）的大小。在矩阵乘法 $Y = XW$ 中，如果某个输入通道的激活值 $X_i$ 极大，那么对应的权重 $W_i$ 的量化误差就会被成百上千倍地放大。
- 工程妥协的终极目标：理论上，把这 1% 保持 FP16，剩下的 99% 压成 INT4 精度最高（混合精度）。但 GPU 底层极度厌恶这种“碎片化”的显存读取。因此，AWQ 的核心任务是：如何在保持 100% 纯 INT4 计算（迎合硬件）的前提下，无损保护这 1% 的精英权重？


## 2. 算法流程与本质思考
AWQ 的做法不是改变数据类型，而是进行等价数学变换：给重要权重乘上一个放大系数 $S$，同时给输入激活值除以 S：Y = XW = (XS^{-1})(SW)。

### 2.1 官方校准流程 (Calibration)
1. **收集统计量**：用一小批数据（约 128 条）跑一遍前向传播，计算每个通道输入激活值的平均绝对值 $s_X$。激活值越大对应的参数越重要。
2. **定义缩放公式**：设定缩放系数 $s_i = (s_{X,i})^\alpha$。
3. **网格搜索 (Grid Search)**：在 $[0, 1]$ 之间遍历 $\alpha$ 的值，针对每一层计算量化后的输出 MSE Loss。MSE loss代表的是用当前参数量化后的误差，越小越好。
4. **确定最优解**：选出让 Loss 最小的 $\alpha$，生成最终的静态缩放矩阵 $S$。
5. **算子融合部署**：把 $SW$ 量化为 INT4，把 $S^{-1}$ 提前乘入前一层 RMSNorm 的权重里，实现推理阶段 0 额外计算开销。

### 2.2 💡 透过现象看本质（底层架构思考）
AWQ并不是硬性截断找出1%的参数不量化，剩下99%的参数量化。他是通过来乘S来决定量化里参数的重要性，一层的W矩阵只有一个Alpha，所以S越大对应的w位置乘的系数越大。这套方案是自适应的，99%的那些参数自动就S就很小嘛，正好对应核心idea。并且在search的时候有mse保证这个误差是足够小的。这个公式天然完成了“论资排辈


## 3. 大模型生态中的广泛应用 (Training vs. Deployment)
在真实的工业界 Pipeline 中，AWQ 不用于训练，而是作为模型上线前的最后一道“瘦身工序”。它与 QLoRA 等技术分工极其明确：

* **第一阶段：微调训练 (Fine-tuning)**
  * **工具**：QLoRA (比如用 NF4 格式加载底座，FP16 训练 LoRA 矩阵)。
  * **目的**：以极低的显存代价，让模型学会特定的业务能力（比如多轮交互或视觉问答）。此时为了保证梯度反向传播的精度，绝对不能用 AWQ。
* **第二阶段：合并与后量化 (Post-Training Quantization, PTQ)**
  * **操作**：把训练好的 LoRA 权重与原模型合并，恢复成一个完整的 FP16 大模型。然后，使用AWQ 算法跑一遍 Calibration，在不重新训练（无反向传播）的情况下，花十几分钟把它压成纯粹的 W4A16 (Weight INT4, Activation FP16) 格式。
* **第三阶段：高并发推理部署 (Inference Deployment)**
  * **工具**：vLLM 或 TensorRT-LLM。
  * **现状**：由于 LLM 的推理是极度 Memory-Bound（显存带宽瓶颈）的，AWQ 打包好的 INT4 权重能让显存读取量减少近 70%。借助底层高度优化的 W4A16 GEMM 算子，AWQ 格式的模型在吞吐量（Throughput）和首字延迟（TTFT）上，是目前工业界高并发 C 端业务的绝对标配。
  * infer的时候参数反量化到fp16和tensor fp16计算。
