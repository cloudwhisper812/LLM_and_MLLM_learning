### MLLM
> 个人总结的话：1.通用任务，更难更专业。2. 细粒度的视觉理解 3. 测试幻觉 4. 测试作为agent的UI操作的能力
#### 数据集
1. 综合与专家知识基准

MMMU (Massive Multi-discipline Multimodal Understanding)
- 目前 MLLM 界的“高考”+“考研”，属于专家级多模态基准。
- 涵盖艺术、工程、科学、医学等学科的大学级别问题。不仅有自然图像，还有大量核磁共振图、乐谱、电路图、化学结构式。
- 它的题极难，大量题目需要极强的交叉学科推理（比如看图算物理题），非常考验预训练阶段（我们之前聊的）高质量图文数据和 Math/Code 数据的厚度。
- 选择题
  
MMBench (体系化能力雷达)
- 评测体系最完整、打分逻辑最严密的基准之一。核心特征： 涵盖了从 L1 到 L3 的 20 种细粒度能力（如：粗粒度感知、逻辑推理、空间关系、属性比较、身份推理等）。
- 大模型做选择题有严重的“位置偏置 (Position Bias)”（比如无脑选 C）。MMBench 会把同一个问题的 A/B/C/D 选项顺序打乱，让模型做多次。只有模型在所有顺序下都能选对，这道题才算得分。 这是面试中非常容易被问到的防作弊细节
- 选择题
  
2. 细粒度与结构化感知
   
DocVQA / ChartQA / InfoVQA (文档与图表理解)
- 密集文本识别、版面分析、数据提取。
- 开放填空/开放/计算
  
MathVista (视觉数学推理)
- 极其依赖模型在 SFT 阶段学到的思维链（CoT）能力。模型不仅要“看懂”辅助线，还要把视觉信息无缝转换为数学逻辑。
- 综合推理

3. 幻觉与对齐评测
   
POPE (Polling-based Object Probing Evaluation)
- 疯狂地问模型 Yes/No 问题，比如“图里有自行车吗？”
- 数据来源于 MSCOCO 等标准数据集，但精髓在于它生成“假问题”的三种采样策略
- Random: 随机问图里没有的物体。Popular: 问数据集中最高频出现的物体（利用模型的统计偏置钓鱼）。Adversarial (对抗): 图里有键盘，但没有鼠标。系统故意问“有鼠标吗？”（利用物体的共现偏置钓鱼）。优秀的模型必须在对抗模式下依然保持极低的 False Positive。
- 只看那个 Yes 或 No
  
HallusionBench： 比ROPE更难
考察点： 专门针对“视觉错觉”和“常识冲突”。图里画了一个红色的苹果，但旁边写着“这是一个香蕉”。考模型是信自己的眼睛（视觉特征），还是信文字（语言特征）。HallusionBench 的数据不是简单的爬虫抓取，而是通过 “原图 + 修改图” 的方式手工打造的。

4. Agentic 与 GUI 操作 (The "Mobile/UI Tracker")

ScreenQA / Mind2Web
- 考察点： 理解手机 App 或网页的 UI 截图。
- 硬核细节 (Grounding)： 模型不仅要回答“购物车按钮在哪”，还要输出精准的 Bounding Box 坐标（比如 [0.15, 0.88, 0.25, 0.95]）。这要求模型在指令微调（SFT）阶段见过了大量由专用 UI 树解析器标注的坐标数据。
